# Dayboard PRD

日期：2026-06-04

## 产品定位

Dayboard 是一个面向 Windows 桌面的轻量日程与任务管理软件。它把日历和待办面板放在桌面上，让用户在不打开完整日历应用的情况下，快速查看今天、这一周和本月安排，并能快速创建任务或日程。

## 目标用户

- 需要在 Windows 桌面上持续看到日程的人，例如独立开发者、学生、远程办公者、项目管理者。
- 同时使用 Google Calendar、Outlook 或本地待办，但觉得完整日历应用过重的人。
- 想要类似轻量版滴答清单体验，但更偏桌面贴片和日历视图的人。

## 核心问题

- 日程信息分散在多个账号和应用里，查看成本高。
- 传统日历应用需要主动打开，无法自然融入桌面工作流。
- 桌面 widget 往往只展示信息，缺少足够顺手的快捷编辑。

## MVP 范围

MVP 先验证“桌面日历贴片 + 本地任务/日程”体验，不在第一版实现真实 Gmail/Outlook 同步。

### 必须有

- 桌面贴片窗口
  - 无边框或弱边框视觉。
  - 支持拖动位置。
  - 记住窗口位置和尺寸。
  - 支持显示/隐藏。
  - 支持固定为桌面面板，默认不占据任务栏主要注意力。
  - 默认尺寸接近半屏工作板，而不是小型信息卡片。
  - 主界面不必常驻显示产品名称，品牌信息可放在设置、托盘或关于页。

- 日历视图
  - 月视图。
  - 周视图。
  - 双周视图。
  - 默认主界面以月视图占满贴片，任务抽屉默认隐藏。
  - 今天高亮。
  - 本地假数据或本地数据展示。
  - 在日历格子或当天时间线中直接显示任务/日程标题。
  - 点击日历中的任务/日程后可编辑。
  - 点击日期格子的空白区域可直接新建任务。
  - 支持把任务/日程拖到其他日期进行重新排期。

- 轻量任务/日程
  - 快速新建。
  - 编辑标题、日期、时间、备注。
  - 标记任务完成。
  - 删除任务或日程。
  - 当日任务抽屉默认折叠，常态下优先保留日历空间。

- 托盘入口
  - 显示/隐藏主面板。
  - 打开设置。
  - 退出应用。

- 设置
  - 开机自启开关，默认关闭。
  - 主题选择：跟随系统、浅色、深色。
  - 面板透明度调节，默认保持可读性。
  - 固定方式选择：固定桌面、普通窗口、置顶。
  - 悬浮工具条位置选择：左侧或右侧。
  - 贴片默认模式选择：月、周、双周。
  - 数据位置说明。

- 本地存储
  - MVP 阶段可先使用 WebView/localStorage 验证体验。
  - 桌面壳稳定后迁移到 SQLite 或等价本地数据库。
  - 配置和窗口状态本地保存。
  - 不默认上传任何数据。

### 暂不做

- Google Calendar 真实同步。
- Outlook/Microsoft 365 真实同步。
- 双向同步冲突解决。
- 团队协作。
- 移动端。
- 插件市场或 widget 模板市场。
- 复杂番茄钟、习惯追踪、项目管理能力。

## 后续同步范围

### Phase 2：Google Calendar 只读同步

- OAuth 登录。
- 拉取事件并缓存到本地。
- 支持手动同步。
- 支持 token 过期处理。
- 覆盖时区、全天事件、重复事件、取消事件。

### Phase 3：双向同步与 Outlook

- 本地新增/编辑/删除同步到远端。
- 冲突检测和用户确认。
- Outlook/Microsoft Graph 接入。
- 多账号颜色区分。

## 隐私和安全原则

- 默认本地优先。
- 不默认收集用户日历数据。
- 不在日志中记录事件正文、OAuth token、邮箱地址等敏感信息。
- token 必须使用系统级安全存储或加密方案。
- 用户可以关闭同步、清除本地缓存、退出账号。

## 成功标准

- 用户无需打开浏览器或完整日历应用，即可在桌面上看清今天和本周安排。
- 默认使用月视图，周和双周由用户主动切换。
- 日历区域应是主视觉和主交互区，任务标题能在日历中直接辨认。
- 从看到空闲时间到创建一条任务的操作足够短。
- 应用常驻桌面时不明显干扰工作。
- MVP 在 Windows 上可稳定启动、退出、隐藏、恢复，并能保留窗口状态。
