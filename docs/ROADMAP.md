# Dayboard Roadmap

日期：2026-06-04

## Milestone 0：项目底座

目标：完成产品边界、技术路线和目录约定。

- 建立 PRD、Roadmap、决策记录和技术调研笔记。
- 确定 MVP 不直接做真实日历同步。
- 确定首选技术栈。
- 初始化应用源码目录。

## Milestone 1：本地桌面贴片原型

目标：验证 Dayboard 是否适合作为常驻桌面面板。

- 创建 Windows 桌面应用骨架。
- 实现主贴片窗口。
- 实现托盘菜单。
- 实现显示/隐藏、退出、窗口位置记忆。
- 使用本地假数据展示月、周、双周视图。
- 接入 OpenDesign 原型方向，形成深色半透明桌面贴片外观。
- 记录窗口层级、透明度、任务栏显示等 Windows 行为。

验收：

- 应用可以在 Windows 上启动。
- 面板可以拖动并记住位置。
- 托盘可以控制显示、隐藏和退出。
- UI 不依赖真实网络账号。

## Milestone 2：本地任务和日程

目标：让 Dayboard 具备日常可用的轻量记录能力。

- 本地存储模型。MVP 先用 WebView/localStorage 验证体验，桌面壳稳定后迁移 SQLite。
- 快速新建任务/日程。
- 编辑、完成、删除。
- 月、周、双周视图联动。
- 拖动任务到其他日期进行重新排期。
- 基础搜索或今日列表。

验收：

- 关闭应用再打开后数据仍保留。
- 常见本地任务操作不需要打开设置页。
- 有基础测试或手动验证记录。

## Milestone 3：Google Calendar 只读同步

目标：验证同步入口和日历数据模型。

- Google OAuth 登录。
- 只读拉取事件。
- 本地缓存。
- 手动同步。
- token 过期处理。
- 覆盖全天事件、重复事件、取消事件、时区。

验收：

- 用户可连接一个 Google 账号。
- 日历事件可展示在 Dayboard 视图里。
- 登出可以清除 token 和缓存。
- 不在日志泄漏敏感数据。

## Milestone 4：多账号与双向同步

目标：从“可看”走向“可编辑”。

- Google Calendar 创建、修改、删除。
- 冲突检测。
- Outlook/Microsoft Graph 接入。
- 多账号颜色和过滤。
- 离线编辑队列。

验收：

- 本地修改不会静默覆盖远端重要变更。
- 用户能清楚看到同步状态和错误。

## Milestone 5：体验打磨

目标：提高长期常驻桌面的舒适度。

- 主题和密度设置。
- 快捷键。
- 更好的空状态和错误状态。
- 自动更新。
- 安装包和发布流程。
