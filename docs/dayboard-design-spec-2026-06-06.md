# Dayboard 本轮界面设计规格

日期：2026-06-06

## 1. 本轮目标

本轮只聚焦一组高保真桌面原型，覆盖以下 3 个界面：

1. 主界面：**左右半屏工作板**
2. 快速编辑 / 新建面板
3. 桌面贴片：**常驻半屏贴片样式**

这轮不做真实 Gmail / Outlook 同步，只表现**账号连接入口状态**，用于验证 Dayboard 作为 Windows 常驻轻任务日历的核心体验。

## 2. 设计定位

Dayboard 不是传统“打开后再操作”的完整日历，而是一个默认贴在桌面的**轻量工作板**。

本轮设计强调 4 件事：

- **看得快**：打开或扫一眼就能看到今天、本周、关键安排
- **改得快**：点空白直接新建，点条目直接编辑，减少跳转
- **常驻不吵**：默认像桌面面板，而不是抢注意力的主应用
- **负担低**：不要求用户先做复杂设置，先支持本地任务管理，再逐步接日历账号

## 3. 选定方案

本轮采用 **修正版方案 C**：

**左右半屏工作板 + 可切换的桌面贴片 + 双周 / 摘要 / 月视图共存**

原因：

- Windows 桌面横向空间足够大，左右半屏更符合常驻桌面的工作板心智
- 双周视图仍然是主界面的核心叙事，适合“看接下来 10–14 天安排”的轻任务节奏
- 桌面贴片需要兼顾总览与低干扰，因此应支持在双周与摘要之间根据宽度和用户意图切换
- 用户额外要求贴片支持月视图，说明贴片不仅是提醒面板，也承担轻量日历总览入口
- 比固定单一形态更接近真实桌面产品，可适配半屏、窄贴边与更紧凑的常驻状态

## 4. 视觉系统

整体方向：**现代极简 / Windows 桌面效率工具**

本轮不把“液态玻璃”作为主风格，而是作为**局部增强层**使用：主界面保持清晰和工具化，玻璃感只出现在贴片外壳、浮层和次级容器上。

### 4.1 颜色

- 主背景：冷灰白 / 微蓝中性色
- 面板底：以清爽实心面为主，贴片与浮层可加入轻微半透明
- 主强调色：清晰蓝，用于主操作、选中态、当前日期
- 次强调色：柔和青绿，用于完成状态、同步状态、稳定反馈
- 辅助色：低饱和灰，用于说明文字、边框、非激活控件

### 4.2 液态玻璃使用原则

- **可用，但只局部使用**
- 主要用于：桌面贴片外壳、快速编辑浮层、少量次级信息容器
- 不用于：日历主体、任务文字区、输入字段主体、长列表区域
- 原则是“**框体有玻璃感，信息层不玻璃化**”
- 目标是增加贴桌面的高级感，而不是牺牲阅读效率去追求炫技

### 4.3 组件气质

- 细边框
- 低阴影
- 12–16px 圆角
- 面板层级轻，不做厚重卡片堆叠
- 默认强调内容区，不强调品牌露出
- 玻璃效果只作为容器质感，不作为主内容层的视觉主角

### 4.4 字体与信息层级

- 字体气质：紧凑、清爽、工具化
- 一级信息：日期、任务标题、核心视图
- 二级信息：时间、日历来源、状态说明
- 三级信息：备注、提示文案、设置解释

## 5. 信息架构

### 5.1 主界面结构

主界面默认是接近屏幕左右半区的桌面工作板，而不是上下堆叠的小组件。

推荐结构：

- **顶部极简时间导航**
  - 当前月份
  - 前后翻页
- **侧边悬浮工具条**
  - 拖动窗口
  - 新建任务
  - 展开 / 收起任务面板
  - 主题预览入口
  - 设置入口
  - 隐藏到托盘
- **中央主区域：双周视图 / 摘要视图 / 月视图容器**
  - 本轮默认以双周视图为主
- **右侧任务与详情面板**
  - 当前日期
  - 当天条目
  - 搜索
  - 快速新建

### 5.2 快速编辑面板结构

用于新建或编辑任务 / 日程：

- 标题
- 日期
- 类型（任务 / 日程）
- 开始时间 / 结束时间
- 日历来源
- 状态
- 备注
- 删除 / 取消 / 保存

目标不是完整表单，而是**最少字段完成一次可用编辑**。

### 5.3 桌面贴片结构

贴片不是独立做成一个“花哨 widget”，而是主界面的常驻形态延伸。

默认原则：

- 半屏贴附
- 弱边框 / 半透明
- 支持 **双周 / 摘要 / 月视图** 三种内容模式
- 可直接勾选完成
- 可直接点空白新增
- 保留账号状态入口，但不展开复杂同步流程

#### 贴片模式定义

1. **双周模式**
   - 适用于较宽的半屏贴附状态
   - 保留双周日历骨架，右侧信息压缩为简短今日事项与快速新增
   - 是最接近主工作板的压缩态

2. **摘要模式**
   - 适用于较窄、低干扰的常驻状态
   - 重点展示今天日期、今日任务、本周接下来 3–5 项
   - 保留一个快速新增入口与一个轻量模式切换入口

3. **月视图模式**
   - 用于用户需要更长跨度排期时快速总览全月
   - 以紧凑月历为主，弱化右侧详情，只保留选中日期摘要
   - 不承担复杂编辑流程，重点是“扫一眼整月安排”

#### 默认切换规则

- 较宽贴片：默认进入 **双周模式**
- 较窄贴片：默认进入 **摘要模式**
- **月视图不作为自适应默认态**，而是用户主动切换后的稳定模式
- 顶部提供一个低显眼度的 segmented control：`双周 / 摘要 / 月` 
- 用户手动切换后，记住上一次选择；除非尺寸变化显著导致当前模式不可用，再回退到最合适模式

#### 贴片中的月视图约束

- 月格子优先显示日期与少量事件标记，不堆满文字
- 选中某一天后，在下方或右侧展示当天 1–3 条核心事项
- 月视图里仍保留“点空白新增”和“点条目进入快速编辑”的短路径
- 月视图需要保持安静，不做传统重型日历软件那种密集工具栏

## 6. 核心交互原则

### 6.1 新建任务

- 点击日历格子空白直接新建
- 点击工具条“+”直接基于当前日期新建
- 新建优先使用弹层，不跳转整页

### 6.2 编辑任务

- 点击日历中的条目直接打开编辑面板
- 点击右侧列表中的条目也可进入编辑
- 编辑后原地保存，保持连续操作感

### 6.3 完成任务

- 在右侧任务列表里一键勾选完成
- 完成态通过颜色和样式弱化，但仍保留可见性

### 6.4 视图切换

- 主界面支持 **双周 / 摘要 / 月视图** 三态切换
- 本轮主工作板仍以 **双周视图** 作为默认核心展示模式
- 摘要视图承担窄态、低干扰和今日焦点浏览
- 月视图保留为完整时间跨度总览入口
- 选中日期的折叠任务面板承担聚焦当天编辑与查看的职责，不再额外暴露日视图主入口
- 贴片中的双周与摘要可根据宽度自适应默认，月视图由用户主动切入
- 默认贴片模式放在设置内选择，并单独记忆

### 6.5 桌面常驻

- 提供固定桌面 / 普通窗口 / 置顶三种模式入口
- 当前原型先展示可见入口与切换反馈
- 真正的 Windows 窗口层级行为留到实现阶段继续验证

## 7. 账号连接策略（本轮）

本轮只做到**入口状态**，不进入真实 OAuth 流程。

位置建议：

- 放在主界面右上角，或日历标题区旁边
- 作为次级系统状态入口存在，不进入主视觉中心

表现建议：

- 小图标 + 简短状态文案
- 默认弱化显示
- hover / 点击后再展开更多动作
- 不使用主按钮色，不做大面积高亮

建议呈现三种状态中的基础两种：

- 未连接账号
- 已连接 1 个账号

必要时可补一个扩展态：

- 连接更多账号

设计目的：

- 让用户理解产品未来会支持 Gmail / Outlook
- 不让同步能力在 MVP 早期压过核心桌面体验
- 保持当前原型范围与 PRD / 决策记录一致
- 让账号能力像 quiet status / utility entry，而不是主流程入口

## 8. 与现有原型的对应关系

结合当前代码状态，已有方向与本轮规格大体一致：

- 已有主工作板结构
- 已有月视图和任务编辑弹层
- 已有快速编辑弹层
- 已有固定方式与工具条位置设置
- 已有本地数据与示例数据重置入口

但从设计规格角度，下一步实现应进一步强化以下几点：

1. **双周视图要真正成为默认主叙事**，而不是只是多种视图之一
2. **账号连接入口状态需要进入主界面信息层级**，而不是缺席
3. **桌面贴片感需要更强**，包括更明确的常驻半屏布局、双周 / 摘要 / 月视图切换逻辑
4. **主界面与贴片的关系要统一**，避免像“一个日历页 + 一个设置页”的组合

## 9. 本轮实现建议

进入实现阶段时，建议优先按以下顺序落地：

### 9.1 第一优先级

- 把默认视图从月视图调整为双周视图
- 在主界面加入账号连接状态入口
- 调整主界面文案与结构，使其更像“桌面工作板”而非普通日历页

### 9.2 第二优先级

- 强化桌面贴片样式
- 实现贴片的 **双周 / 摘要 / 月视图** 模式切换与记忆
- 优化右侧任务面板的摘要与操作节奏
- 让“今日 / 本周接下来几项”更突出

### 9.3 第三优先级

- 打磨交互细节
- 统一颜色语义
- 优化空状态、完成态、选中态和 hover 反馈

## 10. 本轮不进入的范围

以下内容不在这轮设计规格落地范围内：

- 真实 Gmail OAuth
- 真实 Outlook / Microsoft Graph 接入
- 双向同步
- 冲突处理
- 多账号颜色体系
- 复杂项目管理能力
- 移动端适配

## 11. 设计验收标准

本轮设计可视为到位，需满足：

- 一眼能看出这是一个**Windows 桌面常驻轻任务日历**
- 双周视图成为主要认知中心
- 桌面贴片支持双周 / 摘要 / 月视图三态，并且切换逻辑自然
- 新建 / 编辑 / 完成任务路径足够短
- 贴片感明显，但不牺牲可读性
- 账号连接虽然未实现，但入口与未来方向清楚
- 整体气质简洁、现代、安静，不像重型日历软件
